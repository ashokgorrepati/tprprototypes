B2B Use Case: Profile+Event (Multi-Metric Query)

This prototype demonstrates a Profile+Event Multi-Metric query, which produces a result similar to a "Right Outer Join."

Persona

A Sales Director.

Use Case

The Sales Director needs a complete view of all accounts with an open pipeline. They must see *all* accounts with open opportunities, sorted by value, and then append any recent sales activity to that list to spot high-value, un-engaged accounts.

Business Question

"I need a single report of *all* my accounts with 'Open' opportunities, sorted by the total value of those opps. For each account, show me the number of 'Sales Calls' from the last 30 days."

Key Concept: Profile+Event (Multi-Metric Query)
This query mixes scopes. It uses a **Profile-scoped metric** (`Total Open Opp Value`) and an **Event-scoped metric** (`Sales Calls`). When Workspace sees a profile-scoped metric, it builds the table based on the *full profile list* first (all accounts with "Open" opps). It then joins the event data from the panel's date range. Accounts with no matching events (no sales calls) will show 0 for the event metric, rather than being dropped from the table.

Required Joins

  1. `Event Dataset` is joined to the `Account` container.
  2. `Account Lookup` is joined to the `Account` container (Path: `Account_Profile_Join`).
  3. `Opportunity Lookup` is joined twice:
    • Once to the `Opportunity` container (Path: `Event_Opportunity_Join`) for event-based reporting.
    • Once to the `Account` container (Path: `Account_Opportunities_Join`) for profile-based reporting.

How This Query Works (The "Right Join")

This table demonstrates the "multi-metric" or "right join" query behavior, which is driven by the mix of metric scopes and the path-aware filter:

  1. The "Driver": The query is "driven" by two profile-scoped components:
    • The metric Total Open Opp Value (which is opp_value set to "Total Population" scope).
    • The filter Opportunity Status = 'Open'.
    Both of these *must* come from the **`Account_Opportunities_Join`** path to be valid.
  2. The Initial List: The system first queries the `Opportunity Lookup` (using the `Account_Opportunities_Join` path) to find all accounts that have an `status = 'Open'` opportunity. This creates the initial list: Global Tech, Innovate Solutions, and SMB Services. The "Closed-Won" opp for Global Tech is ignored.
  3. The "Join": The Sales Calls metric is "Event-based", so it respects the "Last 30 Days" date range and the `event_type = 'Sales Call'` filter. Workspace joins this event data to the profile-based list.
  4. The Result (Innovate Solutions): This account is on the list because it has a $2M "Open" opp. It had *no 'Sales Call' events in the last 30 days* (only a 'Web Visit'), so it correctly shows 0 for the `Sales Calls` metric. This is the "right join" in action—the account is not dropped.
